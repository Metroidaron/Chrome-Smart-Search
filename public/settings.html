Rules
This form of editing the rules has not been implimented yet, Editing these rules will not modify the functions of the web page. One of each of these rules has been impimented to establish what fields need to utilized and as a scaffold for designing the javascript behind each item. These rules should be added and removed dynamically.
  • Match : Matches user input to a rule, defined by a Regular Expression. User can opt to use a 'Key' to force recognition of this rule, otherwise this rule will follow standard recognition rules.
  • Redirect : Redirects the user to the given url, much like a book mark. This rule is not meant to pass any data to the target website.
  • GET : allows the user to pass a selection of values to the target website (all values predetermined except what the user types to triger this rule, if the user would like to pass multiple values the user needs to utilize the 'Script' rule)
  • POST : Same as the GET Rule, but using POST Values. (More Technical and requires a stronger understanding of the target website)
  • Script : Allows a user to write a custom script to execute when this rule is triggered
